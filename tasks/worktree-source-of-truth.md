# Refactor: git worktree list as source of truth

## Summary

`store.Workspace` currently redundantly persists `TmuxSession`, `WorktreePath`, and `Status` — all of which are deterministically derivable at runtime. This refactor makes `git worktree list` the authority for active workspace state and reduces the store to a metadata-only layer: stable identity (`ID`, `Name`), timestamps, idle-detection state, and `Branch` (see decision below). A new `workspace.Workspace` combined type replaces `store.Workspace` as the public API of the Manager and all downstream consumers. `store.Workspace` becomes an internal implementation detail never exposed beyond the `workspace` package.

**Branch for archived workspaces — decision: keep `Branch` in `store.Workspace` (option c).**
`Delete()` must delete the git branch by name *after* `Archive()` has already removed the worktree, so the name must be persisted somewhere. Storing `Branch` once at create time (it is set by the caller and does not change during normal usage) is simpler than threading it through the `Archive()` return path. For active workspaces `workspace.Workspace.Branch` is overridden from `git worktree list` (authoritative); for archived workspaces it falls back to the stored value. The fields removed are only the ones that can be *computed* without any state — `TmuxSession = cfg.SessionPrefix + Name`, `WorktreePath = filepath.Join(cfg.WorktreeRoot, Name)`, `Status` derived from `ArchivedAt + worktree presence + tmux liveness`. Existing JSON store files with the old fields load cleanly; Go's `json.Unmarshal` silently ignores unknown keys, so no migration code is needed.
